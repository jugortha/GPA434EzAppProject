EzApp

Introduction

La librairie EzApp a été créée pour faciliter le développement d'une application ayant une interface graphique rudimentaire.

Les outils offerts sont simples mais limités. L'objectif étant d'offrir une interface de programmation permettant de créer des petits projets stimulants en langage C++ où l'emphase est mis sur les autres aspects que l'interface utilisateur.

SFML

La librairie EzApp est entièrement basée sur la librairie SFML. En fait, EzApp est une simplification de la libraire SFML. D'ailleurs, le vocabulaire utilisé en est librement inspiré.

Éléments principaux

Voici une brève description des constituants :

  • la librairie utilise le namespace ezapp (tout en minuscule).
  • la classe ezapp::EzApp permet de créer une application :
    • trois fonctions peuvent être substitué permettant de créer une application personnalisée (polymorphisme)
    • la sous-classes ezapp::EzApp::Parameters permet la paramétrisation des éléments importants de l'application
      • taille de la fenêtre
      • titre de la fenêtre
      • icône de la fenêtre
      • niveau d'anti crénelage pour l'affichage
  • les classes permettant la gestion des entrées/sorties:
    • ezapp::Assets : ressources
    • ezapp::Keyboard : clavier
    • ezapp::Mouse : souris (non fonctionnel)
    • ezapp::Screen : l'affichage
    • ezapp::Sound : le son (non fonctionnel)
    • ezapp::Timer : la gestion de temps
  • les classes liées aux graphiques :
    • ezapp::CircleShape : forme circulaire
    • ezapp::PolygonalShape : forme polygonale
    • ezapp::RectangleShape : forme rectangulaire
    • ezapp::TextShape : forme circulaire
  • primitives géométriques :
    • ezapp::Point : représente un point dans l'espace 2d (classe template)
      • PointInt = Point<int>
      • PointReal = Point<float>
    • ezapp::Size : représente une taille dans l'espace 2d (classe template)
      • SizeInt = Size<int>
      • SizeReal = Size<float>
    • ezapp::Rectangle : représente un rectangle dans l'espace 2d (classe template)
      • RectangleInt = Rectangle<int>
      • RectangleReal = Rectangle<float>

Installation et configuration

Sachez que la librairie a été compilée avec VisualStudio 2017 en 64 bits uniquement. Il est donc impossible de l'utiliser sous MAC ou Linux (sans émulateur Windows) ou en mode x86. Par ailleurs, la librairie est compatible avec Visual Studio 2019 mais les indications plus bas sont valides mais se retrouvent à des endroits qui peuvent être différentes.

Les indications ici sont en anglais (désolé, je n'avais qu'une version anglophone lors de la rédaction de ces instructions).

Voici les étapes pour créer un projet utilisant la librairie EzApp dans VisualStudio :

  1. Création d'un projet
    • Menu : File | New | Project...
    • Dans la fenêtre : New Project :
      • Type : Empty project
      • Name : donner un nom à votre projet
      • Location : déterminer le dossier
      • Solution Name : donner un nom à votre solution
    • Prenez en note le nom du projet ainsi que de la solution car pour le reste des indications, nous y ferons référence :
      • [[pro]] : le dossier du projet
      • [[sol]] : le dossier de la solution
      • On remarque d'ailleurs que votre projet se retrouve sous solution : ...\[[sol]]\[[pro]]\.
  2. Créer le point d'entrée du projet :
    • Menu : Project | Add New Items...
    • Dans la fenêtre : Add New Item - [pro] :
      • Type : C++ File (.cpp)
      • Name : le nom que vous voulez (on suggère souvent soit : source.cpp, main.cpp ou préférablement [[pro]].cpp)
    • Dans le fichier ajouté, mettre ce code :
      #include <EzApp.h>
      int WinMain()
      {
      ezapp::EzApp a;
      a.run();
      return 0;
      }
  3. Copie de la librairie dans votre solution :
    • Copier le dossier fourni nommé EzApp dans le dossier de votre solution [[sol]].
    • Vous devriez avoir cette structure de dossiers :
      • [[sol]]
        • EzApp
          • doc
          • header
          • lib
          • resources
        • [[pro]]
  4. Copie de la police de caractères Arial dans votre projet :
    • Copier le fichier fourni nommé Arial.ttf dans le dossier de votre projet [[pro]].
  5. Configuration de VisualStudio :
    • Menu : Project | [[pro]] Properties
    • Dans la fenêtre : [[pro]] Property Pages
      • Étape 1 :
        • Configuration
          • All Configurations
        • Platform
          • x64
        • C/C++ | General | Additional Include Directories : ajouter
          • ..\EzApp\header\
        • C/C++ | Langage | C++ Language Standard : choisir
          • ISO C++ Latest Draft Standard (/std:c++latest)
        • Linker | Input | Additional Dependencies : ajouter
          • ezapp.lib
        • Linker | System | SubSystem : choisir
          • Windows (/SUBSYSTEM:WINDOWS)
      • Étape 2 :
        • Configuration
          • Debug
        • Platform
          • x64
        • Linker | General | Link Library Dependency : ajouter
          • ..\EzApp\lib\Debug\
      • Étape 3 :
        • Configuration
          • Release
        • Platform
          • x64
        • Linker | General | Link Library Dependency : ajouter
          • ..\EzApp\lib\Release\
  6. Faire un test : Exécuter le programme. Vous devriez avoir une fenêtre de 800 x 600 titrée EzApp! avec un arrière pla noir.

Problèmes fréquents :

  • Si vous avez suivi attentivement ces indications et que vous n'arrivez pas à compiler, il est possible que vous n'avez pas mis le projet courant en mode x64.
  • Si l'application compile, est liée et s'exécute sans problème mais crash au démarrage, c'est certainement que vous avez oublié d'ajouter la police de caractères Arial. La police de caractères est une ressource nécessaire à l'exécution de la librairie.

Utilisation

Pour créer une application, il suffit de créer sa propre classe héritant de ezapp::EzApp et de substituer les fonctions virtuelles suivantes :

  • ezapp::EzApp::setup : zone de préparation avant de lancer la boucle d'exécution
  • ezapp::EzApp::updateFromInputs : permet de réagir aux différents évènements et de faire les traitements nécessaires à l'application
  • ezapp::EzApp::updateOutputs : permet de donner du feedback à l'utilisateur via l'écran et les haut parleurs.

Après avoir créé une instance de cette classe, l'appel de la fonction ezapp::EzApp::run démarre l'application. L'application s'exécute tant que l'utilisateur n'a pas cliqué sur le bouton de fermeture (le X en haut à droite) et que la fonction ezapp::EzApp::updateFromInputs retourne vrai.
